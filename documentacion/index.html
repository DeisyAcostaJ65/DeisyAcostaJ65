Documentación

Hugo es una de las herramientas para generar sitios estáticos de código abierto moderno que usa lenguaje de programación GO. Con su increíble velocidad y flexibilidad, Hugo vuelve a hacer divertido la creación de sitios web.

La distribución de los ficheros se muestra a contiuación:

|--archetypes
|--content
|--data
|--public
|--resources
|--themes
|--config.toml

Ficheros

archetypes: Almacena plantillas de Markdown para usarse en el contenido.

content: Archivos con el contenido ordenado en carpetas que se mostrará en el sitio público. El contenido puede estar en formato markdown (archivos .md) o directamente en formato HTML.

data: Contenido adicional para incluir en el sitio web. Esta carpeta contiene archivos en formato TOML, YAML, JSON, incluso CSV u otros, y constituye una especie de base de datos de contenido estructurado y accesible a través de un sistema de objetos y propiedades. Este contenido adicional también puede estar almacenado en content.

public: Contienen el sitio web listo para publicar generado a partir del contenido de /content, los archivos estáticos de /static y según las reglas y las plantillas definidas en el resto de carpetas de Hugo.

resources: Este contiene varios activos como estilos y scripts.

themes: Almacena los temas y estilos de la página estática.

config.toml: Archivo de configuración de Hugo. Puede estar en formato TOML, YAML y JSON.
